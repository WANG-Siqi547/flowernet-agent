【核心组件】
================================================================================

1. Generator（生成模块） - 端口 8002
   - generator.py: 核心生成逻辑（~400 行）
   - main.py: FastAPI 服务（~150 行）
   - Dockerfile: 容器配置

2. Verifier（验证模块） - 端口 8000 
   - verifier.py: 验证逻辑（相关性 + 冗余度）
   - main.py: FastAPI 服务

3. Controller（控制模块） - 端口 8001
   - controler.py: Prompt 优化逻辑（~250 行）
   - main.py: FastAPI 服务

4. Python 客户端库 - flowernet_client.py
   - FlowerNetClient: 简化 API 调用
   - FlowerNetDocumentGenerator: 文档生成助手

================================================================================
【快速开始】
================================================================================

Step 1: 设置 API Key（按你使用的 provider）
   export AZURE_OPENAI_API_KEY="your-key"
   # 或
   export GOOGLE_API_KEY="your-key"

Step 2: 启动所有服务
    cd flowernet-agent
    python3 start_services.py

Step 3: 快速检查
   curl -s http://localhost:8002/health
   curl -s http://localhost:8000/health
   curl -s http://localhost:8001/health

Step 4: 使用客户端生成内容
    from flowernet_client import FlowerNetClient
    client = FlowerNetClient()
    result = client.generate_with_loop(
        outline="主题",
        initial_prompt="请介绍这个主题"
    )
    print(result['draft'])

================================================================================
【工作流程】
================================================================================

输入: 大纲 + 初始 Prompt
  ↓
[Generator] 调用当前 provider 生成 Draft
  ↓
[Verifier] 验证 (相关性 ≥ 0.6 且 冗余度 ≤ 0.7)
  ├─→ ✅ 通过 → 返回结果
  └─→ ❌ 失败 → [Controller] 分析问题并修改 Prompt
       ↓
    (回到 Generator，循环)
       ↓
最多重复 max_iterations 次

================================================================================
【主要 API】
================================================================================

FlowerNetClient 方法：

1. generate(prompt, max_tokens=2000)
   简单生成，不进行验证

2. verify(draft, outline, history=[])
   验证内容

3. generate_with_loop(outline, initial_prompt, max_iterations=3)
   完整循环（生成 → 验证 → 修改 → 循环）

4. health_check()
   检查所有服务状态

FlowerNetDocumentGenerator 方法：

1. generate_document(title, outlines, max_iterations=3)
   生成完整多段落文档

================================================================================
【参数调整】
================================================================================

rel_threshold (相关性阈值，范围 0-1)
  0.3-0.5: 宽松，快速
  0.5-0.7: 标准 ⭐ 推荐
  0.7-0.9: 严格，高质量

red_threshold (冗余度阈值，范围 0-1)
  0.5-0.6: 严格去重
  0.7-0.8: 标准 ⭐ 推荐
  0.8-0.9: 宽松，允许重复

max_iterations (最大循环次数)
  1-2: 快速模式
  3-5: 平衡模式 ⭐ 推荐
  5+: 完美模式

    )

================================================================================
【故障排除】
================================================================================

❌ "API Key not found"
   → 检查当前 provider 对应的 key（如 `AZURE_OPENAI_API_KEY` / `GOOGLE_API_KEY`）

❌ "Connection refused"
   → python3 start_services.py

❌ 生成很慢（30+ 秒）
   → 正常，降低 max_iterations 或 rel_threshold

❌ 验证一直失败
   → 降低 rel_threshold (0.4) 或提高 red_threshold (0.8)

❌ "Port already in use"
   → lsof -i :8000 | grep main.py | kill -9

【性能指标】
================================================================================

单段落生成时间: 20-60 秒（包括 API 调用和验证）
- 第一次迭代: 15-30 秒
